# Supabase Database Guide

This is the single main Supabase reference for the project. Use this file when setting up, reviewing, or migrating the current database.

## 1. Purpose

The application currently uses Supabase PostgreSQL for:

- User login through Supabase Auth.
- Client and status configuration.
- Device type configuration.
- Device inventory records.
- Ongoing testing records.
- Archived deleted records.
- Audit trail movement history.
- Image uploads for ongoing testing package pictures.

## 2. Environment Setup

Create `.env.local` for local development:

```env
# Supabase project URL from Project Settings > API
VITE_SUPABASE_URL=https://your-project.supabase.co

# Public anon key from Project Settings > API
VITE_SUPABASE_ANON_KEY=your-anon-key
```

For Vercel, add the same variables in Project Settings > Environment Variables.

## 3. Deployment Steps

1. Open Supabase SQL Editor.
2. Make sure the SQL Editor role is admin/postgres for schema creation.
3. Run the full SQL script below.
4. Open Supabase Authentication and create user accounts for login.
5. Confirm the `ongoing-testing-images` storage bucket exists and is public.
6. Add the environment variables in Vercel.
7. Run `npm run build` before deployment.
8. Deploy the frontend to Vercel.

## 4. Full Supabase SQL Script

Run this script once for a new Supabase project. The script is written in study mode: schema lines include inline comments, and structural lines have nearby comments explaining why they exist.

```sql
/*
====================================================
Purpose : Create the full database schema for
          Endivio Device Management
Module  : All Modules
Database: Supabase PostgreSQL
Date    : 2026-06-09
====================================================
*/

-- Enable UUID helper functions when future UUID columns are needed.
create extension if not exists pgcrypto;

-- Store clients used by inventory and testing dropdowns.
create table if not exists public.clients (
  id bigint generated by default as identity primary key, -- Unique client row identifier.
  name text not null, -- Full client/company name shown in configuration.
  client_code text not null, -- Short client code shown in inventory/testing records.
  is_active boolean not null default true, -- Controls if the client appears in active dropdowns.
  created_at timestamptz not null default now() -- Date/time when the client was created.
); -- End of clients table definition.

-- Explain the business purpose of the clients table.
comment on table public.clients is 'Stores client records and unique client codes for dropdown selection.';

-- Explain why client_code exists separately from name.
comment on column public.clients.client_code is 'Unique short code displayed in inventory and testing records.';

-- Prevent duplicate client codes regardless of letter casing.
create unique index if not exists clients_client_code_unique
on public.clients (lower(client_code)); -- Index lower(client_code) so duplicate codes are blocked case-insensitively.

-- Store reusable statuses and colors for inventory/testing workflows.
create table if not exists public.statuses (
  id bigint generated by default as identity primary key, -- Unique status row identifier.
  name text not null unique, -- Status label shown to users.
  color text not null default '#4b5563', -- Hex color used for status chips.
  is_active boolean not null default true, -- Controls if the status appears in active dropdowns.
  created_at timestamptz not null default now() -- Date/time when the status was created.
); -- End of statuses table definition.

-- Explain the business purpose of the statuses table.
comment on table public.statuses is 'Stores configurable status labels and colors used by inventory and testing modules.';

-- Store reusable device types for the inventory Device Type selector.
create table if not exists public.device_types (
  id bigint generated by default as identity primary key, -- Unique device type row identifier.
  name text not null unique, -- Device type/model label shown in inventory selection.
  is_active boolean not null default true, -- Controls if the device type appears in active dropdowns.
  created_at timestamptz not null default now() -- Date/time when the device type was created.
); -- End of device types table definition.

-- Explain the business purpose of the device types table.
comment on table public.device_types is 'Stores configurable device type labels used by inventory records.';

-- Prevent duplicate device type names.
create unique index if not exists device_types_name_unique
on public.device_types (lower(name)); -- Index lower(name) so duplicate device types are blocked case-insensitively.

-- Store completed or received device inventory records.
create table if not exists public.device_inventory_items (
  id bigint generated by default as identity primary key, -- Unique inventory record identifier.
  company text, -- Company/client display name retained for reporting.
  client_id bigint references public.clients(id) on delete set null, -- Linked client record.
  raised_by text, -- Person who raised or encoded the inventory item.
  date_received date, -- Date the device was received.
  package_style text, -- Packaging type such as With Box, Plastic, or Paper Bag.
  cst_number text, -- CST reference number.
  ticket_number text, -- Support ticket reference number.
  sn_number text, -- Device serial number.
  device_type text, -- Device model/type.
  with_adapter text, -- Indicates whether the package includes an adapter.
  start_repairing_support date, -- Start date for repairing/support work.
  end_date_support date, -- End date for support work.
  start_qa date, -- QA start date.
  end_date_qa date, -- QA end date.
  status_id bigint references public.statuses(id) on delete set null, -- Linked status record.
  date_delivered date, -- Date the device was delivered or deployed.
  give_to text, -- Person who received the delivered device.
  remarks text, -- Free-form notes and issue details.
  source_ongoing_id bigint, -- Ongoing testing record id when transferred from testing.
  created_at timestamptz not null default now(), -- Date/time when the row was created.
  updated_at timestamptz -- Date/time when the row was last edited.
); -- End of device inventory table definition.

-- Explain the business purpose of the inventory table.
comment on table public.device_inventory_items is 'Stores device inventory records, delivery tracking, QA dates, status, and remarks.';

-- Improve filtering and joins by client.
create index if not exists device_inventory_items_client_id_idx
on public.device_inventory_items(client_id); -- Index the client foreign key for faster client-based reports.

-- Improve filtering and joins by status.
create index if not exists device_inventory_items_status_id_idx
on public.device_inventory_items(status_id); -- Index the status foreign key for faster status filtering.

-- Improve date range filtering in Inventory Records.
create index if not exists device_inventory_items_date_received_idx
on public.device_inventory_items(date_received); -- Index received date for date range filtering.

-- Store devices that are still under testing or repair.
create table if not exists public.ongoing_testing_items (
  id bigint generated by default as identity primary key, -- Unique ongoing testing row identifier.
  client_id bigint references public.clients(id) on delete set null, -- Linked client record.
  date_received date, -- Date the testing item was received.
  package_style text, -- Packaging type of the received device.
  picture_url text, -- Public storage URL for the package image.
  model text, -- Device model under testing.
  with_adapter text, -- Indicates whether the package includes an adapter.
  serial_number text, -- Device serial number.
  status_id bigint references public.statuses(id) on delete set null, -- Linked testing status.
  repair_by text, -- Person assigned to repair.
  test_by text, -- Person assigned to test.
  senior_test_by text, -- Senior tester or reviewer.
  remarks text, -- Testing notes and issue details.
  source_inventory_id bigint, -- Inventory record id when transferred back from inventory.
  created_at timestamptz not null default now(), -- Date/time when the row was created.
  updated_at timestamptz -- Date/time when the row was last edited.
); -- End of ongoing testing table definition.

-- Explain the business purpose of the ongoing testing table.
comment on table public.ongoing_testing_items is 'Stores devices currently under testing, repair, or review.';

-- Improve filtering and joins by client.
create index if not exists ongoing_testing_items_client_id_idx
on public.ongoing_testing_items(client_id); -- Index the client foreign key for faster testing reports.

-- Improve filtering and joins by status.
create index if not exists ongoing_testing_items_status_id_idx
on public.ongoing_testing_items(status_id); -- Index the status foreign key for faster status filtering.

-- Improve date range filtering in Ongoing Testing.
create index if not exists ongoing_testing_items_date_received_idx
on public.ongoing_testing_items(date_received); -- Index received date for date range filtering.

-- Store deleted records so they can be restored later.
create table if not exists public.archived_records (
  id bigint generated by default as identity primary key, -- Unique archive row identifier.
  source_table text not null check (source_table in ('device_inventory_items', 'ongoing_testing_items')), -- Original table where the record came from.
  record_type text not null, -- User-friendly module name.
  record_label text, -- Human-readable record identifier.
  record_data jsonb not null, -- Full row payload needed for restore.
  archived_by uuid default auth.uid(), -- Supabase Auth user who archived the record.
  archived_at timestamptz not null default now() -- Date/time when the record was archived.
); -- End of archived records table definition.

-- Explain the business purpose of archived records.
comment on table public.archived_records is 'Stores deleted inventory and testing records for user-driven restore.';

-- Improve archive page sorting by newest archived record.
create index if not exists archived_records_archived_at_idx
on public.archived_records(archived_at desc); -- Index archived date so newest deleted records load first.

-- Store all user-visible movement and configuration actions.
create table if not exists public.audit_trail (
  id bigint generated by default as identity primary key, -- Unique audit event identifier.
  event_time timestamptz not null default now(), -- Date/time when the event happened.
  module text not null, -- Application module that created the event.
  action text not null, -- Action such as CREATE, UPDATE, DELETE, ARCHIVE, RESTORE, or TRANSFER.
  entity_table text not null, -- Table affected by the event.
  entity_id text, -- Affected record id stored as text for flexibility.
  record_label text, -- Human-readable record identifier.
  actor_id uuid, -- Supabase Auth user id.
  actor_email text, -- Supabase Auth user email.
  summary text, -- User-friendly movement summary for reports.
  before_data jsonb, -- Record snapshot before the change.
  after_data jsonb, -- Record snapshot after the change.
  metadata jsonb -- Extra technical context such as source record ids.
); -- End of audit trail table definition.

-- Explain the business purpose of audit trail records.
comment on table public.audit_trail is 'Stores movement history across inventory, testing, archive, restore, and configuration modules.';

-- Improve required date range filtering in the Audit Trail page.
create index if not exists audit_trail_event_time_idx
on public.audit_trail(event_time desc); -- Index event time because audit generation always requires a date range.

-- Improve investigation of movement for a specific record.
create index if not exists audit_trail_entity_idx
on public.audit_trail(entity_table, entity_id); -- Index table/id pairs for future record movement investigation.

-- Enable RLS for client configuration.
alter table public.clients enable row level security;

-- Enable RLS for status configuration.
alter table public.statuses enable row level security;

-- Enable RLS for device type configuration.
alter table public.device_types enable row level security;

-- Enable RLS for inventory records.
alter table public.device_inventory_items enable row level security;

-- Enable RLS for ongoing testing records.
alter table public.ongoing_testing_items enable row level security;

-- Enable RLS for archived records.
alter table public.archived_records enable row level security;

-- Enable RLS for audit trail records.
alter table public.audit_trail enable row level security;

-- Remove old clients read policy before recreating it.
drop policy if exists "authenticated users can read clients" on public.clients;

-- Allow signed-in users to read clients.
create policy "authenticated users can read clients"
on public.clients for select
to authenticated
using (true);

-- Remove old clients manage policy before recreating it.
drop policy if exists "authenticated users can manage clients" on public.clients;

-- Allow signed-in users to create, update, and delete clients.
create policy "authenticated users can manage clients"
on public.clients for all
to authenticated
using (true)
with check (true); -- Allow authenticated writes when the new client row passes policy checks.

-- Remove old statuses read policy before recreating it.

-- Allow signed-in users to read statuses.

on public.statuses for select
```

## 5. Migration Notes To MSSQL

Use `MSSQL.md` when moving away from Supabase. The important rule is that a React browser app should not connect directly to MSSQL. Build a backend API first, then move the SQL schema.

Suggested migration order:

1. Create MSSQL database and tables from `MSSQL.md`.
2. Export Supabase table data as CSV.
3. Import CSV files into MSSQL staging tables.
4. Validate row counts for every table.
5. Build API endpoints for the frontend.
6. Replace Supabase frontend calls with API calls.
7. Test login/auth replacement.
8. Test inventory, testing, archive, restore, and audit flows.
9. Switch production environment variables.

## 6. Rollback Instructions

Use rollback only after exporting data.

```sql
/*
====================================================
Purpose : Remove Supabase project tables for rollback
Module  : Database Rollback
Database: Supabase PostgreSQL
====================================================
*/

-- Drop child/history tables first because they depend on base records.
drop table if exists public.audit_trail;

-- Drop archived records after audit trail.
drop table if exists public.archived_records;

-- Drop testing records before shared lookup tables.
drop table if exists public.ongoing_testing_items;

-- Drop inventory records before shared lookup tables.
drop table if exists public.device_inventory_items;

-- Drop statuses after dependent records are removed.
drop table if exists public.statuses;

-- Drop device types after inventory text values are already preserved/removed.
drop table if exists public.device_types;

-- Drop clients after dependent records are removed.
drop table if exists public.clients;
```
